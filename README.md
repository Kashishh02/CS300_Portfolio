## Reflection

### What was the problem you were solving in the projects for this course?

The primary problem I was solving in this course was how to efficiently organize, store, and retrieve course information for the ABCU advising program. The goal was to create a system that could load course data, display courses in alphanumeric order, and provide detailed information about individual courses and their prerequisites.

### How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by analyzing different data structures, including vectors, hash tables, and binary search trees. Understanding data structures is important because they directly affect how efficiently a program stores, searches, and retrieves data. By comparing different approaches, I gained a better understanding of how data structures impact performance and scalability.

### How did you overcome any roadblocks you encountered while going through the activities or project?

One challenge was implementing the data structures correctly and ensuring the program produced the required output. I overcame these issues by testing frequently, breaking larger problems into smaller tasks, and debugging my code step by step. This helped me identify errors and improve my problem-solving skills.

### How has your work on this project expanded your approach to designing software and developing programs?

This project taught me the importance of planning before coding. I learned to evaluate multiple solutions before selecting an approach and to consider efficiency, maintainability, and future scalability when designing software.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

Throughout this course, I learned the value of writing organized code with meaningful names, clear structure, and modular design. These practices make programs easier to understand, maintain, debug, and modify in the future.
